# Qt 并行弹丸注入可视化界面设计文档与用户说明书

## 1. 文档目的

本文档用于说明 `visualization/` 目录下 Qt 可视化前端的设计思路、模块划分、交互流程、数据流和用户使用方式。本文档同时作为：

- 前端界面的设计说明
- 并行仿真可视化工具的用户说明书
- 后续功能扩展和维护的参考文档

本文档面向以下读者：

- 项目开发者
- 界面维护者
- 使用该工具进行本地仿真和调试的研究人员

---

## 2. 功能目标

该前端界面的目标不是替代底层物理仿真器，而是在现有 `TransportSimulator + PelletSimulator` 基础上提供一个本地桌面调试入口，用于快速配置、运行、观察并保存并行弹丸注入仿真结果。

核心目标如下：

- 提供本地 Qt 桌面界面，便于实验室环境下直接使用
- 支持并行 batch 仿真，统一控制注入参数
- 支持通过 `ITER.py` 配置文件驱动仿真
- 支持常用输出量选择和自定义扩展输出量
- 仅实时显示第一个 batch 的可视化结果，降低界面复杂度
- 保存全部 batch 的完整结果，便于离线分析
- 保持调试友好，能够在终端同步输出关键状态值

---

## 3. 系统定位与范围

### 3.1 系统定位

该界面是一个本地桌面仿真控制台，定位为“调试优先”的可视化工具，而不是面向公网部署的 Web 系统。

选择 Qt 的主要原因如下：

- 与当前 Python/JAX/Torch/Torax 的本地执行模型更匹配
- 文件路径选择、运行控制、线程管理、结果保存更直接
- 不需要引入前后端分离架构，维护成本更低
- 本地实验环境使用更稳定

### 3.2 当前实现范围

当前实现已覆盖：

- 参数配置
- 并行仿真启动与停止
- 标量曲线可视化
- 一维径向剖面可视化
- 二维极向截面热力图可视化
- 运行状态显示
- 日志显示
- 结果保存

当前不包含：

- 时间滑条回放
- 多 batch 同时可视化
- 远程 Web 访问
- 内置参数编辑器
- 训练型 SB3 交互控制

## 4. 总体架构设计

### 4.1 架构概览

前端采用“Qt 主界面 + 后台仿真线程 + 绘图组件 + 数据模型”的结构。

主要模块如下：

- `visualization/app.py`
  - 主窗口
  - UI 布局与交互控制
  - 信号连接与状态刷新
- `visualization/simulator_worker.py`
  - 后台仿真线程
  - 调用 `TransportSimulator` 与 `PelletSimulator`
  - 负责预览数据生成与结果打包
- `visualization/plotting.py`
  - 标量曲线、一维剖面、二维截面绘图组件
- `visualization/data_models.py`
  - 配置项定义
  - 默认字段
  - 几何信息读取
  - 结果保存路径管理
- `visualization/theme.py`
  - 界面样式表
  - 深色主题配置

> 插图占位：图 4-1 系统总体架构图

### 4.2 运行流程

1. 用户在主界面输入仿真参数
2. 主界面构造 `SimulationSettings`
3. 用户点击 `Start`
4. `SimulationWorker` 在后台线程中启动仿真
5. 每隔若干步生成预览数据并通过 Qt signal 发回主线程
6. 主线程更新界面中的曲线、剖面、状态与日志
7. 仿真完成后将完整结果保存在内存中
8. 用户点击 `Save` 将结果写入 `visualization/runs/<run_id>/`

## 5. 界面设计

### 5.1 视觉风格

界面采用深色与灰色为主的科技风格，目标是让长时间运行时的视觉刺激更低，同时保持一定的仪器感和工程感。

设计特点如下：

- 主背景为深色近黑
- 面板使用灰蓝色边框和低对比度底色
- 重点按钮使用蓝色和红色区分主操作与停止操作
- 指标信息使用高亮浅色文字突出显示
- 图表区域使用暗背景，适合高亮曲线和热力图显示

> 插图占位：图 5-1 主界面整体风格截图

### 5.2 三段式布局

界面采用三段式布局：

- 左侧：参数配置区
- 中间：可视化展示区
- 右侧：状态与日志区

这样设计的原因如下：

- 左侧适合放置输入控制项，操作集中
- 中间区域留给图表，占据最大视觉面积
- 右侧用于监控仿真状态和阅读日志，不干扰主图表

> 插图占位：图 5-2 三段式布局标注图

### 5.3 左侧参数配置区

左侧区域包含以下功能模块。

#### 5.3.1 配置文件选择

用户可通过文件选择框指定 `ITER.py` 路径。当前设计只要求指定路径，不支持在界面中直接编辑配置文件。

默认路径：

```bash
config/ITER.py
```

#### 5.3.2 并行参数设置

用户可以设置：

- `Parallel batch`
  - 并行仿真数量
  - 范围：`1` 到 `2048`
  - 默认值：`4`
- `Simulation steps`
  - 仿真总步数
  - 默认值：`10000`
  - 对应 `10 s`，因为默认 `dt = 1 ms`

#### 5.3.3 弹丸注入参数设置

用户可以设置：

- `Injection interval`
  - 注入间隔，单位 `ms`
  - 默认值：`100`
- `Pellet velocity`
  - 注入速度，单位 `m/s`
  - 默认值：`300`
- `Pellet thickness`
  - 弹丸厚度，单位 `mm`
  - 默认值：`4.0`

设计约束：

- 所有 batch 共用同一组注入参数
- 当前版本不支持每个 batch 单独配置注入参数

#### 5.3.4 输出量选择

用户可以通过两种方式指定输出量：

- 预置多选列表
- 自定义输入框扩展字段名

界面会对输出量进行分类：

- 标量量
- 向量量

如果用户没有选择任何向量量，系统会自动加入 `n_e` 作为默认向量量，以保证可视化区域始终可用。

#### 5.3.5 控制按钮

控制按钮包括：

- `Start`
- `Stop`
- `Save`

按钮逻辑：

- 运行开始后，`Start` 禁用，`Stop` 启用
- 仿真结束后，`Save` 启用
- 未完成仿真时不能直接保存结果

### 5.4 中间可视化区

中间区域是主显示区域，包含三个核心可视化组件。

#### 5.4.1 标量监控图

用于显示随时间变化的标量量，例如：

- `Q_fusion`
- `H98`
- `q_min`
- `fgw_n_e_volume_avg`
- `n_e_core`
- `T_e_core`
- `T_i_core`

设计说明：

- 支持多条曲线同时显示
- 每条曲线使用独立颜色
- 横轴为 step
- 纵轴为该量的数值

#### 5.4.2 一维径向剖面图

用于显示第一个 batch 在当前时刻的径向 profile，例如：

- `n_e`
- `T_e`
- `T_i`
- `n_i`
- `q_face`
- `s_face`

设计说明：

- 横轴为归一化半径 `rho`
- 纵轴为选中物理量的数值
- 通过下拉框切换当前显示的向量场

#### 5.4.3 二维极向截面图

用于显示第一个 batch 的二维截面预览，采用热力图方式展示 profile 在反应堆截面上的分布。

设计说明：

- 当前版本不做真实三维几何重建
- 使用径向 profile 结合几何参数构造极向截面色图
- 颜色表示物理量强度
- 当前支持与一维 profile 联动切换

#### 5.4.4 仅显示第一个 batch

为了控制界面复杂度和刷新成本，当前界面只可视化第一个 batch 的结果。

其余 batch：

- 不在界面中显示
- 但仍完整参与仿真
- 并在保存时写入最终的 `pkl` 文件

## 6. 模块设计说明

### 6.1 主窗口模块

主窗口模块位于：

```bash
visualization/app.py
```

负责内容如下：

- 构建整体界面布局
- 管理参数输入控件
- 启动和停止后台线程
- 接收预览数据并刷新图表
- 管理保存按钮状态
- 显示日志和状态信息

### 6.2 仿真线程模块

后台仿真线程位于：

```bash
visualization/simulator_worker.py
```

负责内容如下：

- 创建 `TransportSimulator`
- 创建 `PelletSimulator`
- 构造批量注入输入
- 推进仿真步进
- 提取标量与向量结果
- 构造第一个 batch 的预览数据
- 通过 Qt signal 将状态回传主线程
- 在终端打印关键状态值，便于区分是真实 `NaN` 还是界面显示问题

### 6.3 绘图模块

绘图模块位于：

```bash
visualization/plotting.py
```

负责内容如下：

- 标量曲线绘制
- 一维剖面绘制
- 二维截面绘制
- 图形刷新与清空
- `matplotlib colorbar` 生命周期管理

### 6.4 数据模型与保存模块

位于：

```bash
visualization/data_models.py
```

负责内容如下：

- 定义默认字段与字段分类
- 读取 `ITER.py` 中的几何参数
- 定义 `SimulationSettings`
- 组织运行结果保存路径
- 将结果写出为 `json + pkl`

### 6.5 主题模块

位于：

```bash
visualization/theme.py
```

负责内容如下：

- Qt 样式表配置
- `pyqtgraph` 全局样式配置
- 深色主题统一设置

## 7. 数据流设计

### 7.1 输入数据流

输入来源包括：

- 用户在界面中设置的仿真参数
- `ITER.py` 中的配置参数
- 底层物理模拟器的默认行为

### 7.2 运行时数据流

运行时数据流如下：

1. 主线程生成 `SimulationSettings`
2. 后台线程读取设置并初始化仿真器
3. 每个 step 调用 `env.step(...)`
4. 获取等离子体 profile
5. 同步到 `PelletSimulator`
6. 计算下一步注入位置、宽度、速率
7. 提取：
   - 标量量
   - 向量量
   - 状态量
8. 生成预览 payload
9. Qt 主线程接收 payload 并刷新界面

### 7.3 输出数据流

最终输出包括：

- 终端打印信息
- Qt 日志面板信息
- `run_metadata.json`
- `run_results.pkl`

保存目录如下：

```bash
visualization/runs/<run_id>/
```

## 8. 线程与性能设计

### 8.1 为什么使用后台线程

仿真计算本身较重，如果直接在 Qt 主线程中运行，会导致界面卡死、无响应、无法停止。

因此将仿真执行放入 `QThread`，实现以下效果：

- 主界面保持响应
- 允许实时刷新图表
- 支持点击 `Stop`
- 日志和状态可以持续更新

### 8.2 当前性能策略

当前版本采用以下策略控制界面负担：

- 只预览第一个 batch
- 预览刷新不是每一步都刷新，而是隔若干步发送一次
- 所有 batch 的完整结果只在后台累积，不在前端全部展开显示

### 8.3 当前性能边界

当以下条件同时出现时，内存和刷新成本会显著上升：

- `batch` 很大
- `steps` 很长
- 选择了较多向量字段

因此建议：

- 调试时优先使用较小 batch
- 长时大批量运行后尽快保存并清理界面状态

## 9. 文件输出设计

### 9.1 输出目录

默认输出目录：

```bash
visualization/runs/
```

每次保存会生成独立运行目录：

```bash
visualization/runs/<run_id>/
```

### 9.2 输出文件

#### 9.2.1 `run_metadata.json`

记录内容包括：

- 配置路径
- batch 数量
- 仿真步数
- 注入参数
- 实际执行步数
- 耗时
- 运行速度
- 几何参数
- 预览字段信息

#### 9.2.2 `run_results.pkl`

记录内容包括：

- 全部 batch 的标量结果
- 全部 batch 的向量结果
- 时间轴
- `rho_axis`
- 字段标签
- 运行 metadata

### 9.3 可视化与保存关系

需要特别说明：

- 界面上只显示第一个 batch
- 文件中保存的是全部 batch 的结果

这意味着：

- 界面用于在线监控
- `pkl` 用于离线分析

## 10. 用户说明书

### 10.1 环境要求

建议使用项目约定的 conda 环境：

```bash
conda activate Nuclear
```

依赖安装：

```bash
python3 -m pip install -r requirement.txt
```

### 10.2 启动方式

在项目根目录执行：

```bash
conda activate Nuclear
python3 visualization/app.py
```

> 插图占位：图 10-1 程序启动后初始界面截图

### 10.3 基本操作流程

#### 步骤 1：选择配置文件

在左侧 `ITER config path` 中选择配置文件路径。

默认情况下使用：

```bash
config/ITER.py
```

#### 步骤 2：设置并行数量与仿真步数

设置：

- `Parallel batch`
- `Simulation steps`

建议初次测试时使用：

- `batch = 1` 或 `4`
- `steps = 100` 或 `1000`

#### 步骤 3：设置弹丸注入参数

设置：

- `Injection interval`
- `Pellet velocity`
- `Pellet thickness`

建议初次测试值：

- `Injection interval = 100 ms`
- `Pellet velocity = 300 m/s`
- `Pellet thickness = 4.0 mm`

#### 步骤 4：选择输出量

在输出量列表中勾选希望记录和展示的字段。

如果需要额外字段，可在 `Custom outputs` 中输入，以逗号分隔。

示例：

```text
q95, q_face
```

#### 步骤 5：启动仿真

点击 `Start` 启动仿真。

运行后可以观察：

- 中间图表是否刷新
- 右侧状态量是否更新
- 日志区是否持续输出
- 终端是否打印关键状态值

#### 步骤 6：停止仿真

若需要提前中止，点击 `Stop`。

注意：

- `Stop` 是请求停止，不一定在当前 step 的瞬间立即中断
- 通常会等待当前 step 完成后退出

#### 步骤 7：保存结果

仿真结束后点击 `Save`。

保存后结果写入：

```bash
visualization/runs/<run_id>/
```

### 10.4 如何理解界面结果

#### 10.4.1 标量图

用于看整体趋势，例如：

- 聚变增益是否上升或下降
- 核心参数是否稳定
- 注入前后系统是否出现明显响应

#### 10.4.2 一维剖面图

用于看当前时刻径向 profile 的形状，例如：

- 中心与边缘的温度差异
- 密度峰值是否明显
- 注入是否改变 profile 形状

#### 10.4.3 二维截面图

用于快速观察当前量在反应堆截面上的空间强度分布。

该图更偏可视化表达，不等价于完整的高保真几何模拟图。

### 10.5 终端调试说明

为了便于区分“仿真本身产生 `NaN`”还是“界面显示错误”，程序会在预览时向终端打印关键状态值，例如：

```text
[Visualization preview] step=1/100 Q_fusion=... fGW=... n_e_core=...
```

判断原则：

- 如果终端打印也是 `nan`，说明仿真结果本身异常
- 如果终端是正常数值，而界面显示为 `NaN`，说明前端显示链路存在问题

### 10.6 常见问题

#### 问题 1：界面打开后没有响应

可能原因：

- 仿真参数过大
- 初次加载模型较慢
- 依赖尚未正确安装

建议：

- 先用 `batch=1`、`steps=10` 做 smoke test
- 检查终端输出

#### 问题 2：右侧状态显示 `NaN`

排查方式：

- 看终端中的 `[Visualization preview] ...` 输出
- 若终端也是 `nan`，说明仿真结果异常
- 若终端正常，则需要检查前端显示逻辑

#### 问题 3：保存按钮无法点击

可能原因：

- 仿真尚未完成
- 线程尚未正确结束
- 当前没有有效结果对象

#### 问题 4：二维图重复刷新时报错

当前版本已经对 `matplotlib colorbar` 的重复销毁问题做了处理。如果再次出现类似问题，应优先检查是否运行的是旧进程。

### 10.7 使用建议

建议使用方式如下：

- 开发调试：小 batch、小 steps、少量输出字段
- 物理观察：重点看 `Q_fusion`、`fGW`、`n_e_core`、`T_e_core`、`T_i_core`
- 离线分析：运行结束后保存 `pkl` 再做进一步处理

## 11. 已知限制

当前版本存在以下限制：

- 仅支持单个 `ITER.py` 配置路径输入，不支持界面内编辑
- 所有 batch 共用同一组注入参数
- 只显示第一个 batch 的可视化结果
- 无时间滑条和历史回放
- 大 batch 和长时间运行时，内存占用会上升
- 某些底层库会在终端输出警告信息，这些并不一定表示界面逻辑错误
